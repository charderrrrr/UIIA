# UIIA UIIA ШКВАЛ

Умный генератор нагрузки с автоматическим анализом результатов. Поддерживает HTTP, TCP и WebSocket протоколы. Вдохновлён мемным котом «uiia uiia».

![UIIA](wwwroot/img/uiia-cat.gif)

## Быстрый запуск в браузере

[![Open in GitHub Codespaces](https://github.com/codespaces/badge.svg)](https://codespaces.new/charderrrrr/UIIA)

## Возможности

- **Три протокола:** HTTP, Raw TCP, WebSocket
- **Режимы нагрузки:** постоянный RPS, линейный рост
- **Режимы атаки:**
  - Стандартный — честные HTTP-запросы
  - Обход блокировок — мимикрия под браузер со случайными заголовками и путями
  - HTTP Flood — заливка случайными данными
  - Slowloris — удержание соединений (TCP)
- **Датасеты:** загрузка CSV-файла с реальным сценарием трафика и настраиваемым масштабом времени
- **Автоматический анализ:**
  - Поиск точки отказа (RPS → критическая задержка)
  - Расчёт безопасного лимита RPS
  - Линейная регрессия с оценкой достоверности (R²)
  - Графики: RPS от времени, задержка от времени, регрессионный анализ
- **Автономный отчёт:** HTML с вшитыми графиками и стилями, открывается на любой машине
- **Фоновый саундтрек:** кот орёт «uiia uiia» пока идёт тест

## Установка и запуск

### Требования

- [.NET 10 SDK](https://dotnet.microsoft.com/download) или выше

### Клонирование

```bash
git clone https://github.com/username/uiia-uiia-shkval.git
cd uiia-uiia-shkval
```

### Запуск

```bash
dotnet run
```

Приложение запустится на `http://localhost:5002`.

## Использование

### Быстрый старт

1. Откройте `http://localhost:5002`
2. Укажите URL цели (например, `http://localhost:8080/api`)
3. Выберите протокол и параметры нагрузки
4. Нажмите **ЗАПУСТИТЬ ТЕСТ**
5. Дождитесь завершения и откройте отчёт

### Конфигурация цели

| Параметр | Описание |
|----------|----------|
| URL цели | Полный адрес тестируемого сервиса |
| Протокол | HTTP, Raw TCP или WebSocket |
| Длительность | Продолжительность теста в секундах |

### Параметры нагрузки

| Параметр | Описание |
|----------|----------|
| Режим | `Постоянный RPS` — фиксированная нагрузка, `Линейный рост` — плавное увеличение от мин. до макс. |
| Мин. RPS | Начальная нагрузка (запросов в секунду) |
| Макс. RPS | Конечная нагрузка |
| Таймаут | Максимальное время ожидания ответа в миллисекундах |

### Режимы атаки

| Режим | Протокол | Описание |
|-------|----------|----------|
| Стандартный | HTTP | Обычные GET/POST запросы |
| Обход блокировок | HTTP | Случайные User-Agent, Referer, пути, куки — имитация легитимного трафика |
| HTTP Flood | HTTP | Отправка случайных данных в URL или теле запроса |
| Slowloris | TCP | Медленное удержание соединений для исчерпания пула подключений сервера |

### Датасеты

Датасет — это CSV-файл с колонками:

| Колонка | Тип | Описание |
|---------|-----|----------|
| `time_ms` | long | Время запроса в миллисекундах от начала сценария |
| `method` | string | HTTP-метод (GET, POST и т.д.) |
| `endpoint` | string | Путь эндпоинта (например, `/api/users`) |
| `body_file` | string | Имя файла с телом запроса (опционально) |

Пример `traffic.csv`:

```csv
time_ms,method,endpoint,body_file
0,GET,/api/status,
150,GET,/api/users,
300,POST,/api/users,create_user.json
450,GET,/api/status,
600,GET,/api/users,
```

Параметр `Масштаб времени` ускоряет или замедляет воспроизведение. Например, при `2.0` датасет проиграется в два раза быстрее.

### Заголовки

Можно добавить произвольные HTTP-заголовки, которые будут отправляться с каждым запросом. По умолчанию добавлен `Content-Type: application/json`.

## Отчёт

После завершения теста генерируется HTML-отчёт, содержащий:

- **Параметры теста** — все настройки запуска
- **Анализ:**
  - **Точка отказа** — при каком RPS задержка достигает критических 5000 мс
  - **Рекомендуемый лимит RPS** — безопасная нагрузка с задержкой до 200 мс
  - **Достоверность модели** — коэффициент детерминации R²
- **Графики:**
  - RPS от времени
  - P50 задержка от времени
  - Регрессия: зависимость задержки от нагрузки
- **Таблица метрик** — первые 100 запросов с временем, задержкой, статусом

Отчёт полностью автономен — все графики встроены в HTML как base64, стили вшиты в `<style>`. Можно открыть на любом компьютере без веб-сервера.

## Ограничения

- Максимальный RPS: **10 000**
- Одновременный запуск: **только один тест за раз**
- Отчёт содержит первые **100 записей** метрик
- Датасет поддерживает только CSV с кодировкой UTF-8
- Slowloris работает только через TCP

## Музыка

Кнопка в левом верхнем углу включает/выключает фоновое воспроизведение звука «uiia uiia cat». Во время теста музыка включается автоматически для создания рабочей атмосферы. Громкость: 40%.
